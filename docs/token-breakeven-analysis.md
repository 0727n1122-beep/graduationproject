## 인사이트

### 왜 단문은 토큰이 늘어나는가?
Claude가 모호한 표현을 제거하고 구조화된 프롬프트로 재작성할 때,
명확한 요구사항과 제약조건을 추가하면서 오히려 길어지는 현상 발생.

### 왜 장문은 토큰이 줄어드는가?
장문 프롬프트에는 filler word, 중복 표현, 흩어진 요구사항이 많아
이를 제거하고 구조화하면 실제 정보 밀도가 높아지면서 토큰 절감 발생.

### 서비스 방향성
- 단순 토큰 절감보다 **재프롬프팅 루프 감소**가 핵심 가치
- 장문 프롬프트 사용자 (복잡한 요청을 하는 비전공자)가 주 타겟
- 멘토 피드백: "큰 서비스 관점에서 봤을 때 유의미하게 동작하는 서비스"
